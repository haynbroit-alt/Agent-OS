Execution Graph OS

AI execution runtime with full observability and safe tool execution.
Every decision is a graph traversal — not a black box.

context → planner → policy → executor → memory → observer

Each node is a registered module with a declared contract. Each edge is a logged transition. Type :graph to inspect live.

In action

egos — execution graph os
$ pip install -e . copy
$ ollama pull llama3.2 copy
$ egos copy

Why it exists

AI tools can plan. But you cannot see or control what happens.
Execution Graph OS fixes that: every action is an explicit edge, logged, auditable, and replayable. You always know exactly what ran and why.

Features

Execution graph model
Nodes are modules. Edges are transitions. Conditions are evaluated against global state.
Dry-run mode
Plan any action without executing. See estimated cost, blocked status, and score before committing.
LLM bypass
After 3 consistent successes, learned patterns skip the LLM entirely. Gets faster over time.
Safe tool execution
Shell commands run against an allowlist. Blocked commands are logged, not silently dropped.
Counterfactual reasoning
Ask what would have happened with a different action against real past episodes.
Self-diagnosis
Behavioral drift, contradiction detection, failure root causes — rendered as a structured report.
Decision log
Every decision appended to JSONL with full traversal path. Greppable. Pipeable. Permanent.
Local-first
Runs on Ollama. Zero cloud dependency at runtime. Your data stays on your machine.

REPL commands

:graphVisualize the execution graph (Mermaid + JSON)
:obsHealth snapshot: success rate, reward trend, calibration
:diagnoseFull behavioral self-analysis
:dry <query>Plan without executing
:cf <action>Counterfactual for last query
:auditPolicy audit — prune dead and overgeneralized rules
:driftBehavioral drift report over time
:registryAll registered modules and their declared contracts
